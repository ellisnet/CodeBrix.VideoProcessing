CodeBrix.VideoProcessing is a fully managed, cross-platform FFmpeg/FFprobe
wrapper for .NET. It is a port of the open-source FFMpegCore library, with the
open-source Instances process-wrapper library vendored directly into it (so the
CodeBrix.VideoProcessing.MitLicenseForever NuGet package has no runtime
dependency on a separate "Instances" package). All incorporated source has been
re-namespaced under CodeBrix.VideoProcessing.* and adapted to the CodeBrix
family conventions (target .NET 10.0, nullable reference type annotations
removed, implicit usings made explicit, file-scoped namespaces).

Both incorporated projects are licensed under the MIT License, which is
compatible with this project's MIT License. The full text of each upstream
license is reproduced below, as required by the MIT License.

The in-memory image bridge in the CodeBrix.VideoProcessing.Imaging namespace
(src/CodeBrix.VideoProcessing/Imaging/) has a design adapted from FFMpegCore's
image-extension projects (FFMpegCore.Extensions.SkiaSharp and
FFMpegCore.Extensions.System.Drawing.Common, part of the FFMpegCore repository
credited in entry 1 below, MIT). Rather than copying their SkiaSharp /
System.Drawing.Common code, it re-expresses the same three operations (in-memory
snapshot, image-as-video-frame, poster-plus-audio) purely in terms of
CodeBrix.Imaging, taken as a package reference (CodeBrix.Imaging.Apache-
LicenseForever, Apache-2.0 - compatible with this project's MIT license). It
therefore inherits the FFMpegCore MIT attribution below and introduces no
additional third-party source.

              FFMpegCore.Extensions.* projects were NOT incorporated)

What was incorporated:
  The complete source of the main "FFMpegCore" library project. Every .cs file
  is present under src/CodeBrix.VideoProcessing/ and carries a
  `//was previously: FFMpegCore.<...>;` provenance comment on its namespace line.

Modifications made during the port:
  - Namespaces renamed from FFMpegCore[.*] to CodeBrix.VideoProcessing[.*].
  - References to the Instances library renamed to
    CodeBrix.VideoProcessing.Instances[.Exceptions] (see entry 2).
  - Target framework narrowed to net10.0.
  - Nullable reference-type annotations ('?' on reference types) and the
    null-forgiveness '!' operator removed (CodeBrix family: nullable disabled).
  - Implicit usings disabled; the SDK's implicit global usings added explicitly.
  - The runtime "System.Text.Json" package reference was dropped, as
    System.Text.Json ships in the .NET 10 shared framework.
  - The assembly-level InternalsVisibleTo attribute now targets
    CodeBrix.VideoProcessing.Tests.
  - No functional/behavioral source changes were made.

Note: the file src/CodeBrix.VideoProcessing/FFMpeg/SnapshotArgumentBuilder.cs
carries an upstream top-of-file header reading "Copyright (c) Microsoft. All
Rights Reserved. Licensed under the MIT license." That header was present in the
FFMpegCore source and has been preserved verbatim.
